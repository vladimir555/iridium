Классы

class  iridium::testing::Tester
 Класс-одиночка, отвечающий за регистрацию, управление и запуск всех тестов в рамках фреймворка. Он собирает тестовые случаи, определенные с помощью макроса TEST, и предоставляет метод run, обычно вызываемый функцией main, сгенерированной IMPLEMENT_TEST_MAIN. Tester может фильтровать тесты на основе аргументов командной строки и организует тесты, возможно, в древовидную структуру для выполнения. Подробнее...

Макросы

#define CONCAT_IMPL(x, y)
 Внутренний макрос для прямого склеивания токенов с использованием оператора препроцессора ##. Это служебный макрос, аналогичный макросам в concat.h, но определенный локально для макросов этого файла. Он не раскрывает свои аргументы перед конкатенацией.
#define MACRO_CONCAT(x, y)
 Макрос, выполняющий конкатенацию токенов с предварительным раскрытием своих аргументов, используя CONCAT_IMPL. Гарантирует, что если x или y сами являются макросами, они будут раскрыты перед склеиванием их результатов.
#define TEST(name)
 Определяет тестовый случай и регистрирует его в Tester singleton. Этот макрос генерирует уникальный класс с именем Test_##name, который наследует от iridium::testing::ITest и iridium::testing::UnitTest. Экземпляр этого класса создается автоматически, и его конструктор регистрирует тест в Tester::instance(). Пользователь должен предоставить тело метода void Test_##name::run() сразу после вызова этого макроса.
#define ASSERT_3(left, condition, right)
 Макрос утверждения для бинарных условий (например, равенство, неравенство, меньше чем). Он вычисляет condition(left, right) и, если результат ложный, сообщает о провале теста, используя функциональность базового класса UnitTest, включая выражение и номер файла/строки.
#define ASSERT_2(func, exception)
 Макрос утверждения для проверки того, выбрасывает ли данный вызов функции определенное исключение. Он выполняет func в контексте, который перехватывает исключения типа exception. Если ожидаемое исключение не выброшено или выброшено другое исключение, сообщается о провале.
#define ASSERT_1(is_true)
 Макрос утверждения для проверки истинности булева условия. Если is_true вычисляется как ложь, сообщается о провале теста.
#define ASSERT(...)
 Макрос-переключатель, который диспетчеризует вызов в ASSERT_1, ASSERT_2 или ASSERT_3 на основе количества предоставленных аргументов. Это позволяет использовать единое имя ASSERT для различных типов утверждений.
#define IMPLEMENT_TEST_MAIN()
 Генерирует функцию main() для исполняемого файла тестов. Эта функция main инициализирует Tester singleton и вызывает его метод run(), передавая аргументы командной строки. Это вспомогательный макрос для быстрой настройки исполняемого файла средства запуска тестов.
