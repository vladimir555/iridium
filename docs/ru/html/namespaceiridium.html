Подробное описание

Определяет интерфейсы для управления контекстом ввода-вывода и именованными каналами.

Включает заголовочный файл с платформо-специфичной реализацией процесса. Фактически включаемый файл (например, "platform/linux/process.h", "platform/windows/process.h") определяется макросом PLATFORM_HEADER, который разрешается на основе обнаруженной операционной системы и архитектуры во время компиляции. Ожидается, что этот заголовочный файл определит platform::CProcessStream.

Определяет класс для разбора и представления универсальных идентификаторов ресурсов (URI).

Определяет основные интерфейсы для операций потокового ввода-вывода, включая чтение, запись и свойства потока.

Определяет интерфейс для менеджера сессий.

Определяет интерфейс для обработчика обобщенного протокола ввода-вывода.

Определяет интерфейсы, связанные с каналоподобными механизмами ввода-вывода, с акцентом на доступ к потокам и передачу данных.

Определяет интерфейс для мультиплексора событий ввода-вывода.

Определяет класс CFileStreamWriter, конкретную реализацию для файловых потоков, ориентированных на запись.

Определяет класс CFileStreamReader, конкретную реализацию для файловых потоков, предназначенных только для чтения.

Определяет класс CFileStream, конкретную реализацию для файловых потоков ввода-вывода.

Определяет класс CFileCache, файловую реализацию интерфейса ICache.

Определяет вспомогательные функции проверки утверждений для низкоуровневых вызовов файлового API. Эти функции оборачивают результаты файловых операций и выбрасывают исключения при обнаружении ошибок.

Платформо-независимый диспетчер подключения для низкоуровневых реализаций файлового API. Этот заголовочный файл использует макрос (PLATFORM_HEADER) для подключения соответствующего платформо-специфичного заголовочного файла file_api.h (например, для Unix, Windows), который содержит фактические объявления низкоуровневых операций файловой системы.

Объявляет служебные функции для операций с файловой системой.

Определяет шаблонный класс FileValue, который синхронизирует значение с файлом.

Определяет интерфейсы для операций с файловыми потоками, включая чтение, запись и получение статуса файла.

Определяет интерфейс для обобщенного кэша строк.

Определяет интерфейс для управления контекстами ввода-вывода.

Подключает платформо-специфичный заголовочный файл, содержащий фактические объявления файлового API. Макрос PLATFORM_HEADER разрешается в путь к файлу file_api.h, находящемуся в подкаталоге, соответствующем текущей целевой платформе (например, iridium/io/fs/implementation/platform/unix/file_api.h или iridium/io/fs/implementation/platform/windows/file_api.h).
