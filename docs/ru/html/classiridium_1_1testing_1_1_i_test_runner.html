Подробное описание

Представляет общий результат выполнения тестового прогона. Эта структура обычно сериализуется в/из файла (например, XML, JSON) и содержит список результатов для отдельных тестовых случаев.

Представляет результат отдельного тестового случая в рамках общих результатов тестирования. Каждый элемент в этом списке соответствует одному выполненному тесту.

Путь или уникальный идентификатор тестового случая.

Сообщение об ошибке, если тест не пройден. Эта строка обычно пуста, если тест пройден.

Любой стандартный вывод, захваченный от тестового случая во время его выполнения.

Интерфейс для средства запуска тестов. Средство запуска тестов отвечает за обнаружение, выполнение тестов (которые организованы в древовидной структуре) и формирование отчета о результатах.

Определения типов

◆ INodeTest

typedef parsing::INodeType<ITest *> iridium::testing::ITestRunner::INodeTest

Определяет общие элементы интерфейса для ITestRunner, такие как typedef-ы умных указателей (TSharedPtr, TWeakPtr и т.д.).

Псевдоним типа для специализированного INodeType, который хранит сырой указатель на экземпляр ITest в качестве своего значения. Используется для представления тестового случая или набора тестов в иерархической структуре узлов, которую средство запуска тестов может обходить и выполнять.

Методы

◆ run()

virtual TResult iridium::testing::ITestRunner::run ( INodeTest::TSharedPtr const & node_test)
pure virtual

Запускает тесты, определенные в предоставленной структуре узлов или под ней.

Аргументы
node_testРазделяемый указатель на корневой узел набора тестов или структуры отдельного тестового случая. Средство запуска обычно обходит это дерево узлов для поиска и выполнения экземпляров ITest.
Возвращает
Объект TResult, содержащий результаты всех выполненных тестов.

Замещается в CTestRunnerFork, CTestRunnerRaw, iridium::testing::implementation::CTestRunnerFork и iridium::testing::implementation::CTestRunnerRaw.
