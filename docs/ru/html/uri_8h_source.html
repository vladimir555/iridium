22// Examples of URI formats understood by this parser:
23// [IPC]://[/usr/bin]/[ls] - protocol + path
24// [FILE]://[/usr/bin]/[ls] - protocol + path + host
25// [PROCESS]://[/usr/bin]/[ls] [dir1] [dir2] - protocol + path + host + arguments
26// [HTTPS]://[hostname.com][/dir1/dir2][?arg1=1&arg2=2] - protocol + host + path + arguments
27// [HTTPS]://[hostname.com][:80][/dir1/dir2][?arg1=1&arg2=2] - protocol + host + port + path + arguments
28// [MYSQL]://[user:password]@[hostname.com][:80][/db_name] - protocol + user + password + host + port + path
29
30// Note on address components:
31// host + port + path = address
32// host + path = address
33// path + host = address (less common, depends on scheme)
