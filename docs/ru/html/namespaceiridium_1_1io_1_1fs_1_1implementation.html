Подробное описание

Платформо-независимый диспетчер подключения для низкоуровневых реализаций файлового API. Этот заголовочный файл использует макрос (PLATFORM_HEADER) для подключения соответствующего платформо-специфичного заголовочного файла file_api.h (например, для Unix, Windows), который содержит фактические объявления низкоуровневых операций файловой системы.

Определяет класс CFileStreamWriter, конкретную реализацию для файловых потоков, ориентированных на запись.

Определяет класс CFileStreamReader, конкретную реализацию для файловых потоков, предназначенных только для чтения.

Определяет класс CFileStream, конкретную реализацию для файловых потоков ввода-вывода.

Определяет класс CFileCache, файловую реализацию интерфейса ICache.

Определяет вспомогательные функции проверки утверждений для низкоуровневых вызовов файлового API. Эти функции оборачивают результаты файловых операций и выбрасывают исключения при обнаружении ошибок.

Подключает платформо-специфичный заголовочный файл, содержащий фактические объявления файлового API. Макрос PLATFORM_HEADER разрешается в путь к файлу file_api.h, находящемуся в подкаталоге, соответствующем текущей целевой платформе (например, iridium/io/fs/implementation/platform/unix/file_api.h или iridium/io/fs/implementation/platform/windows/file_api.h).

Проверяет указатель FILE* из стандартной операции ввода-вывода C и выбрасывает исключение при ошибке. Используется для функций типа fopen, где возвращаемое значение nullptr указывает на ошибку. Если result равен nullptr, выбрасывается std::runtime_error (или аналогичное специфичное для Iridium исключение), включающее предоставленное сообщение и информацию о системной ошибке (например, из errno).

Аргументы
resultУказатель FILE* для проверки.
messageПользовательское сообщение для включения в исключение, если result равен nullptr.
Возвращает
Исходный указатель result, если он не nullptr.
Исключения
std::runtime_error(или аналогичное), если result равен nullptr.
