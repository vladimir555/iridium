Detailed Description

A concrete sink implementation for logging messages to a file, with support for file rotation. This sink writes formatted log messages to a specified file path and can rotate the log file based on strategies like daily rotation or rotation on each initialization.

Parameters
levelThe minimum severity level of events that this sink will process.
pathThe base path for the log file. Depending on the rotation strategy, date/time stamps might be appended to this path.
rotationThe file rotation strategy to use. Defaults to TRotation::UNKNOWN (no rotation or rotation handled by other means).
