Detailed Description

Concrete implementation of ISessionManager. Manages I/O sessions, typically for network connections. It integrates a multiplexer for handling multiple streams, a context manager for associating state with streams, and worker threads for processing events and I/O operations.

void iridium::io::implementation::CSessionManager::manage ( IStreamPort::TSharedPtr const & stream_port,
IProtocol::TSharedPtr const & protocol )
overridevirtual

Macro used to define common interface elements (e.g., virtual destructor).

Instructs the session manager to begin managing a new session based on the provided stream/port and protocol. This method is typically called when a new connection is accepted or a new communication channel is established. The manager will then be responsible for the lifecycle of this session, including event handling and data processing as defined by the protocol on the given stream/port.

Parameters
streamPortA shared pointer to an IStreamPort (or a similar I/O channel abstraction like IStream) representing the communication endpoint for the session.
protocolA shared pointer to an IProtocol that will handle the data communication logic for this session.
Exceptions
iridium::Exceptionor similar if the manager is unable to start managing the session.
