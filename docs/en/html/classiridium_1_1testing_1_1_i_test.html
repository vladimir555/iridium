Detailed Description

Interface for a runnable test case. Concrete test classes should implement this interface to define a specific test. The run() method encapsulates the test logic, including any assertions. Test runners can then execute collections of ITest instances.

Defines common interface elements for ITest, such as smart pointer typedefs (TSharedPtr, TWeakPtr, etc.).

Runs the test case. Implementations of this method should contain the actual test logic and assertions. Any exception thrown during the execution of run() is typically caught by a test runner and interpreted as a test failure.
