IConnector::TSharedPtr iridium::db::createConnector ( config::TDatebase const & config)

Creates a database connector instance based on the provided configuration. This function acts as a factory to instantiate a concrete IConnector implementation (e.g., a MySQL connector, PostgreSQL connector) suitable for the given configuration.

Parameters
configThe database configuration object (config::TDatebase) specifying connection parameters. The Type field in the config determines which type of connector is created.
Returns
A shared pointer (IConnector::TSharedPtr) to an IConnector instance. Returns a null pointer or throws an exception if a connector cannot be created for the given configuration.
Exceptions
iridium::db::Exceptionor other exceptions on failure to create a connector.

Creates a database connector instance from a URI. This is a convenience function that first calls makeConfig(uri) to create a configuration object from the URI, and then calls createConnector(config) with that configuration.

Returns

◆ makeConfig()

config::TDatebase iridium::db::makeConfig ( io::URI const & uri)

Creates a database configuration object from a URI. Parses the given URI and extracts database connection parameters (like type, host, port, user, password, database name) to populate a config::TDatebase object.

Parameters
uriThe URI containing database connection information. The URI scheme might determine the database type (e.g., "mysql://", "postgres://").
Returns
A config::TDatebase object populated with parameters from the URI.
Exceptions
iridium::db::Exceptionor other exceptions if URI parsing fails or scheme is unsupported.
