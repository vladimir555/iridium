Detailed Description

A concrete implementation of IInitializable that manages a collection of other IInitializable objects. This class allows grouping multiple initializable components and managing their lifecycle (initialization and finalization) as a single unit. It ensures that all managed objects are initialized in the order they were provided and finalized in the reverse order.

iridium::pattern::implementation::CInitializer::CInitializer ( TArgs const &... items)

Defines standard implementation helpers for CInitializer, such as a static create method for shared pointer instantiation.

Constructs a CInitializer with a variable number of IInitializable items. The items are stored internally and will be initialized and finalized by this CInitializer instance.

Exceptions
std::runtime_errorif any of the provided `items` is a nullptr.

Initializes the m_items list with the provided pack of shared pointers to IInitializable objects. It iterates through the items to ensure none are null, throwing std::runtime_error if a null item is found.

void iridium::pattern::implementation::CInitializer::finalize ( )
overridevirtual

Finalizes all managed IInitializable objects in the reverse order they were added. Attempts to finalize all items even if some finalizations fail (exceptions from one finalize call do not prevent subsequent finalize calls, though the first exception might be propagated after all attempts). Resets an internal flag to indicate that finalization has been performed.
