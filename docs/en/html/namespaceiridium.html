Detailed Description

Defines interfaces for I/O context management and named pipe management.

Includes the platform-specific process implementation header. The actual file included (e.g., "platform/linux/process.h", "platform/windows/process.h") is determined by the PLATFORM_HEADER macro, which resolves based on the detected operating system and architecture during compilation. This header is expected to define platform::CProcessStream.

Defines a class for parsing and representing Uniform Resource Identifiers (URIs).

Defines core interfaces for I/O stream operations, including reading, writing, and stream properties.

Defines the interface for a session manager.

Defines the interface for a generic I/O protocol handler.

Defines interfaces related to pipe-like I/O mechanisms, focusing on stream access and data transmission.

Defines the interface for an I/O event multiplexer.

Defines the CFileStreamWriter class, a concrete implementation for write-oriented file streams.

Defines the CFileStreamReader class, a concrete implementation for read-only file streams.

Defines the CFileStream class, a concrete implementation for file-based I/O streams.

Defines the CFileCache class, a file-backed implementation of the ICache interface.

Defines assertion helper functions for low-level file API calls. These functions wrap results from file operations and throw exceptions if errors are detected.

Platform-agnostic include dispatcher for low-level file API implementations. This header uses a macro (PLATFORM_HEADER) to include the appropriate platform-specific file_api.h header (e.g., for Unix, Windows), which contains the actual declarations of low-level file system operations.

Declares utility functions for file system operations.

Defines a template class FileValue that synchronizes a value with a file.

Defines interfaces for file stream operations, including reading, writing, and retrieving file status.

Defines the interface for a generic string cache.

Defines an interface for managing I/O contexts.

Includes the platform-specific header file containing actual file API declarations. The PLATFORM_HEADER macro resolves to the path of the file_api.h file located in the subdirectory corresponding to the current target platform (e.g., iridium/io/fs/implementation/platform/unix/file_api.h or iridium/io/fs/implementation/platform/windows/file_api.h).
