Detailed Description

Concrete implementation of the IContextManager interface. Manages the lifecycle of multiple IContext instances, associating them with streams and handling their creation, acquisition, release, and removal. It uses a mutex for thread-safe operations on its internal collections.

IContext::TSharedPtr iridium::io::implementation::CContextManager::acquireContext ( Event::TSharedPtr const & event,
IMultiplexer::TSharedPtr const & multiplexer )
overridevirtual

Acquires a context associated with an event's stream, potentially creating it if it doesn't exist. If a context for the event's stream exists, it is returned. If not, and the event is an Event::NEW_STREAM, a new context is created using the event's stream and protocol. The acquired context is marked as "acquired" to prevent concurrent modification or premature removal.

Parameters
eventThe event whose stream is used to find or create a context.
multiplexerThe multiplexer to associate with the stream if a new context is created and the stream needs to be managed.
Returns
A shared pointer to the acquired or newly created context, or nullptr if no context could be associated or created.
