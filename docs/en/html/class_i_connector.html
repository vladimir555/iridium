Detailed Description

Interface for a database connector. Defines the contract for classes that provide connectivity to a database and allow for query execution. It inherits from iridium::pattern::IInitializable, suggesting that connectors need to be initialized before use (e.g., with configuration).

vector< string > iridium::db::IConnector::makeFields ( TRows const & rows)
static

A utility function to extract unique field (column) names from a dataset. It iterates through the first row of the provided TRows (if not empty) and collects all distinct column names.

Parameters
rowsThe dataset (vector of TRow) from which to extract field names.
Returns
A vector of strings, where each string is a unique field name found in the dataset. The order might depend on the map iteration order of the first row.

◆ sendQuery()

virtual TRows iridium::db::IConnector::sendQuery ( std::string const & query)
pure virtual

Sends a query string to the database for execution. This is a pure virtual method that must be implemented by concrete connector classes. It's responsible for executing SQL queries, DML, DDL, or other database commands.

Parameters
queryThe SQL query or database command to be executed.
Returns
A TRows object containing the results of the query. For DML/DDL statements that do not return a dataset, the returned TRows might be empty or contain information like affected row counts (see FIELD_NAME_AFFECTED_ROWS).
Exceptions
iridium::db::Exceptionor derived classes if a database error occurs.
