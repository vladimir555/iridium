23// Examples of URI formats understood by this parser:
24// [IPC]://[/usr/bin]/[ls] - protocol + path
25// [FILE]://[/usr/bin]/[ls] - protocol + path + host
26// [PROCESS]://[/usr/bin]/[ls] [dir1] [dir2] - protocol + path + host + arguments
27// [HTTPS]://[hostname.com][/dir1/dir2][?arg1=1&arg2=2] - protocol + host + path + arguments
28// [HTTPS]://[hostname.com][:80][/dir1/dir2][?arg1=1&arg2=2] - protocol + host + port + path + arguments
29// [MYSQL]://[user:password]@[hostname.com][:80][/db_name] - protocol + user + password + host + port + path
30
31// Note on address components:
32// host + port + path = address
33// host + path = address
34// path + host = address (less common, depends on scheme)
