Подробное описание

Интерфейс для запускаемого тестового случая. Конкретные тестовые классы должны реализовывать этот интерфейс для определения конкретного теста. Метод run() инкапсулирует логику теста, включая любые утверждения (assertions). Средства запуска тестов затем могут выполнять коллекции экземпляров ITest.

Методы

◆ run()

virtual void iridium::testing::ITest::run ( )
pure virtual

Определяет общие элементы интерфейса для ITest, такие как typedef-ы умных указателей (TSharedPtr, TWeakPtr и т.д.).

Запускает тестовый случай. Реализации этого метода должны содержать фактическую логику теста и утверждения. Любое исключение, выброшенное во время выполнения run(), обычно перехватывается средством запуска тестов и интерпретируется как провал теста.
