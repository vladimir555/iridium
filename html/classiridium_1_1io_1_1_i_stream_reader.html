Подробное описание

Интерфейс для читаемых потоков ввода-вывода. Определяет контракт для потоков, из которых можно читать данные. Наследуется виртуально от IStream, чтобы обеспечить возможность комбинации с IStreamWriter в IStreamPort.

Методы

◆ read()

virtual Buffer::TSharedPtr iridium::io::IStreamReader::read ( size_t const & size = DEFAULT_BUFFER_SIZE)
pure virtual

Макрос, используемый для определения общих элементов интерфейса.

Читает данные из потока размером до указанного. Поведение зависит от состояния потока:

  • Возвращает nullptr при достижении конца файла (EOF) или при неожиданном закрытии потока.
  • Возвращает пустой (но не null) Buffer, если данные в данный момент недоступны (например, в неблокирующем режиме).
  • Если size равен 0, может вернуть ранее накопленный внутренний буфер, если такой механизм существует в реализации.
    Аргументы
    sizeМаксимальное количество байт для чтения. По умолчанию DEFAULT_BUFFER_SIZE. Нулевой размер может иметь специальное значение (см. описание).
    Возвращает
    Умный указатель на Buffer, содержащий прочитанные данные. См. примечания для интерпретации nullptr или пустого Buffer.

Замещается в iridium::io::fs::implementation::CFileStream, iridium::io::implementation::CStreamReaderBuffer и iridium::io::implementation::CStreamReaderList.
